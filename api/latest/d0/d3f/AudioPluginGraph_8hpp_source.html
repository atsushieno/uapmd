15 // AudioPluginGraph is supposed to abstract away how it internally routes audio plugin instances.
16 // It is supposed to `process()` audio and UMP inputs. In that sense, it is a track on a DAW alike.
17 //
18 // It can process nodes in a simple linear way. As a minimum requirement, it must support
19 // a linear chain of nodes, by "add" and "remove" functions.
20 // Advanced DAG topology editing lives in AudioPluginFullDAGraph.
21 //
22 // Note that it is NOT to represent an entire audio graph that is supposed to involve audio device nodes.
23 // That should be handled by a sequencer, which is out of the scope of this class (not even of this library).
24 //
25 // Usually an audio plugin instance shares its lifetime with `AudioPluginNode` when it is being added to the graph,
26 // but the instance is "owned" by the `AudioPluginHostingAPI` implementation.
27 // `appendNodeSimple()` takes a delegate function called when the instance is being removed by `removeNodeSimple()`
28 // or whatever that removes `AudioPluginNode`, to help this mechanism.
